## Critical (must fix)
- `lua_nf_to_lua_contract_derive.lua:83` / `spongejit_lua_ssa.asdl:471` - `SETLIST` is the worst shallow/fake-looking case. It is accepted as `LuaNF.SetListWrite(table, narray, start)`, but unlike `FieldWrite`/`ArrayWrite`, contract derivation does not kill table/barrier facts, and the ASDL/LalinOut shape does not carry the source value range/values being stored. This is not honest as a semantic table write; it is only honest if explicitly modeled as a protocol operation. Concrete improvement: either lower to `SetListProtocolExit(id, pc, table, first_value_slot, nvalues, start, projection)` with a LalinOut `setlist` tag, or make it a real bulk write shape like `BulkArrayWrite(table, start, values: TValue*, barrier_payload)` and derive barrier/table invalidations. Reject variable/multivalue count forms until represented.
